## Solid

https://crazy-crafters.gitlab.io/red-maple/development/solid/overview/#/

- S : Single responsability
- O (Open/closed) : Robot -> Rover peut avancer, le drone peut voler
- L : Liskov substitution : 
  
      Je demande a un Robot de m'apporter le café, mais je suis en haut d'un escalier donc le Rover ne pourra pas le faire.
      C'est bien beau de savoir bouger, il faut aussi savoir comment on bouge, la façon dont on bouge.

- I (interface segregation) : Plusieurs interfaces plutot qu'une
- D (dependency inversion) : Dependre des abstraction, pas de implémentations




https://itexpert.fr/blog/principe-substitution-liskov/
